Positioned Element

1Ô∏è‚É£ Static Positioning (Default)

The element follows the normal document flow, where content is arranged either as block-level or inline elements‚Äîsimilar to a word processor for easy readability. Block-level elements stack on top of each other. Inline elements are placed next to each other. The flow layout ensures elements do not overlap and try to maintain their own space without intruding on others.

2Ô∏è‚É£ Relative Positioning

This behaves similarly to static positioning, but with added flexibility. The element remains in the normal document flow, but you can move it using top, right, left, and bottom without affecting other elements' positions. It does not occupy extra space‚Äîwhen it overlaps with another element, it stacks on top. You can use z-index to control its stacking order. Additionally, the original space is preserved, and the element establishes a new coordinate system based on where it was initially placed in the document flow.

3Ô∏è‚É£ Absolute Positioning

The element is removed from the normal document flow, meaning no space is reserved for it, and other elements behave as if it does not exist in the layout. However, it still appears visually, stacking on top of subsequent elements. An absolutely positioned element establishes a new coordinate system relative to its nearest ancestor that has a position other than static. If a positioned ancestor exists, the element is positioned relative to it. If no positioned ancestor is found, it is positioned relative to the viewport (i.e., the entire page).

4Ô∏è‚É£ Fixed Positioning

When you apply position: fixed; to an element, it is positioned relative to the viewport, meaning it stays fixed in place even when the user scrolls the page. The element is removed from the normal document flow, so it does not occupy space, and other elements behave as if it does not exist in the layout. Unlike absolute, which is relative to a positioned ancestor, a fixed element is always relative to the viewport. This makes it ideal for sticky headers, floating buttons, or navigation bars that need to stay visible at all times.
